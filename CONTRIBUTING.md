# Guía de Contribución

Gracias por tu interés en contribuir a BibliotecaUI!

## Proceso de Contribución

### 1. Fork del Repositorio
Haz un fork de este repositorio a tu cuenta de GitHub.

### 2. Clona tu Fork
```bash
git clone https://github.com/TU_USUARIO/Componentes_web.git
cd Componentes_web
```

### 3. Crea una Rama
```bash
git checkout -b feature/nombre-de-tu-feature
```

### 4. Realiza tus Cambios
- Escribe código limpio y bien documentado
- Sigue las convenciones de código existentes
- Asegúrate de que tu código funcione correctamente

### 5. Commit de tus Cambios
```bash
git add .
git commit -m "Descripción clara de los cambios"
```

### 6. Push a tu Fork
```bash
git push origin feature/nombre-de-tu-feature
```

### 7. Crea un Pull Request
- Ve a tu fork en GitHub
- Click en "Compare & pull request"
- Describe claramente qué cambios realizaste y por qué
- Espera la revisión y aprobación

## Normas de Contribución

### Componentes Nuevos
- Deben incluir TypeScript
- Deben ser responsivos
- Deben seguir el estilo visual del proyecto
- Deben incluir documentación clara

### Código
- Usa TypeScript para type safety
- Sigue las convenciones de nombres existentes
- Comenta código complejo cuando sea necesario
- No incluyas dependencias innecesarias

### Commits
- Mensajes claros y descriptivos
- En español o inglés
- Usa presente: "Agrega componente" no "Agregado componente"

## Revisión de Pull Requests

Todos los Pull Requests serán revisados por **Nicolás Vargas** (@Nico0900).

El proceso puede tomar algunos días. Se pueden solicitar cambios antes de la aprobación.

## Código de Conducta

- Sé respetuoso con otros colaboradores
- Acepta críticas constructivas
- Enfócate en mejorar el proyecto
